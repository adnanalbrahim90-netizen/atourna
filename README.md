# عطورنا — نظام إدارة مبيعات العطور والبخور

مشروع ويب جاهز للنشر على **Netlify مجاناً** بكامل ميزاته (تسجيل مبيعات، فواتير PDF، مخزون، إحصائيات، مستخدمين وصلاحيات، نسخ احتياطي، شعار الشركة).

---

## ✅ البيانات الآن مشتركة ومتزامنة بين كل الأجهزة (عبر Supabase)

هذا المشروع مربوط بقاعدة بيانات سحابية مجانية (**Supabase**). بمجرد ضبط بيانات الاتصال (خطوة واحدة تحتاج 5 دقائق أدناه)، أي مبيعة أو تعديل مخزون يسجّله بائع من جواله يظهر تلقائياً عند المدير وباقي البائعين على أجهزتهم (يتحدّث كل ~4 ثوانٍ).

إذا لم تضبط بيانات Supabase، سيعمل التطبيق تلقائياً بنظام تخزين محلي على كل جهاز بشكل منفصل (بدون مزامنة) — يعني الإعداد التالي **اختياري لكن ضروري لتحقيق مزامنة حقيقية بين البائعين**.

---

## 🗄️ إعداد Supabase (خطوة واحدة، 5 دقائق، مجاني بالكامل)

### 1) أنشئ مشروع Supabase
1. اذهب إلى [supabase.com](https://supabase.com) وأنشئ حساباً مجانياً (يمكن بحساب GitHub مباشرة).
2. اضغط **New project**، اختر اسماً مثل `atourna`، وكلمة مرور لقاعدة البيانات (احفظها)، واختر أقرب منطقة (Region) مثل Frankfurt أو Mumbai.
3. انتظر دقيقة حتى يجهز المشروع.

### 2) أنشئ جدول التخزين
1. من القائمة الجانبية اضغط **SQL Editor → New query**.
2. افتح ملف `supabase.sql` الموجود في هذا المشروع، انسخ محتواه كاملاً، والصقه في المحرر.
3. اضغط **Run**. سترى رسالة نجاح — تم إنشاء الجدول المشترك.

### 3) احصل على مفاتيح الاتصال
1. من القائمة الجانبية: **Project Settings → API**.
2. انسخ قيمتين:
   - **Project URL** (تشبه `https://xxxx.supabase.co`)
   - **anon public key** (مفتاح طويل)

### 4) أضف المفاتيح إلى Netlify
1. في Netlify: **Site settings → Environment variables → Add a variable**.
2. أضف متغيرين:
   - `VITE_SUPABASE_URL` = رابط المشروع الذي نسخته
   - `VITE_SUPABASE_ANON_KEY` = المفتاح العام الذي نسخته
3. اذهب إلى **Deploys → Trigger deploy → Deploy site** لإعادة البناء بالمتغيرات الجديدة.

بعد اكتمال النشر، افتح الرابط من أي جهاز — كل الأجهزة الآن تشارك نفس البيانات فعلياً.

> ⚠️ ملاحظة أمان: مفتاح `anon public key` مفتاح عام مخصص للاستخدام من المتصفح أصلاً (وليس سرياً بنفس درجة كلمة مرور قاعدة البيانات)، لكنه يمنح أي شخص يملك رابط موقعك صلاحية قراءة/تعديل بيانات هذا الجدول تحديداً (لا شيء آخر في حسابك). هذا مقبول لتطبيق داخلي بسيط. إذا احتجت لاحقاً حماية أقوى (تسجيل دخول حقيقي عبر Supabase Auth بدل كلمات المرور المخزّنة داخل الجدول)، أخبرني لأجهزها لك.

## 🚀 طريقة النشر على Netlify (بدون أي خبرة برمجية)

### الخطوة 1: رفع المشروع إلى GitHub (مجاني)
1. أنشئ حساب مجاني على [github.com](https://github.com) إذا ليس لديك.
2. اضغط **New repository**، اختر اسم مثل `atourna-app`، اجعله **Public** أو **Private**، ثم **Create repository**.
3. اضغط **uploading an existing file** وارفع كل ملفات ومجلدات هذا المشروع (اسحبها وأفلتها كاملة كما هي).
4. اضغط **Commit changes**.

### الخطوة 2: ربط المشروع بـ Netlify (مجاني)
1. أنشئ حساب على [netlify.com](https://netlify.com) — الأسهل: اضغط **Sign up with GitHub**.
2. من لوحة التحكم اضغط **Add new site → Import an existing project**.
3. اختر **GitHub** وامنح الصلاحية، ثم اختر مستودع `atourna-app`.
4. Netlify سيكتشف تلقائياً إعدادات البناء (موجودة في ملف `netlify.toml`):
   - Build command: `npm run build`
   - Publish directory: `dist`
5. اضغط **Deploy site** وانتظر دقيقة أو دقيقتين.
6. ستحصل على رابط مجاني مثل: `https://atourna-xxxx.netlify.app` — يمكنك لاحقاً تغييره لاسم أوضح من **Site settings → Change site name**، أو ربط دومين خاص بك.

## 📱 تثبيت الأيقونة على الموبايل (Android / iOS)

بعد فتح رابط الموقع من متصفح الجوال:

**Android (Chrome):**
القائمة (⋮) ← "إضافة إلى الشاشة الرئيسية / Add to Home screen" ← تأكيد.

**iPhone (Safari):**
زر المشاركة (Share) ← "إضافة إلى الشاشة الرئيسية / Add to Home Screen" ← تأكيد.

ستظهر أيقونة **عطورنا** (زجاجة عطر) على الشاشة الرئيسية وتفتح كتطبيق مستقل بدون شريط المتصفح.

## 💻 (اختياري) تشغيل المشروع محلياً على جهاز كمبيوتر

يتطلب تثبيت [Node.js](https://nodejs.org) أولاً:

```bash
cp .env.example .env   # ثم ضع فيه بيانات مشروع Supabase الخاص بك
npm install
npm run dev       # لتجربة المشروع محلياً
npm run build     # لإنشاء نسخة الإنتاج في مجلد dist
```
